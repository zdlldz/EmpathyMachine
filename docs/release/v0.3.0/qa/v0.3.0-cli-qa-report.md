# CLI QA Report: v0.3.0

## 1. Overview
The `mm-cli` was used as the primary verification tool for the v0.3.0 backend refactor and organization features. This report confirms 100% functional parity between the CLI and the GraphQL API.

## 2. Test Cases & Results

### VFS Ingestion & Deduplication
- **Test:** Ingest a folder (`references/MediaMachine`) multiple times.
- **Result:** First ingestion correctly creates nodes and collections. Subsequent ingestions return `count: 0` (Deduplication) without errors.
- **Status:** ✅ PASS

### Automatic Collection Creation
- **Test:** Ingest a directory and verify that a Virtual Collection is created with the same name.
- **Result:** `taxonomy list` confirms creation of "icons" and "MediaMachine" collections with correct slugs.
- **Status:** ✅ PASS

### Metadata & Rating Operations
- **Test:** Set a star rating via `metadata set-rating` and retrieve via `metadata get`.
- **Result:** Successfully set rating to `5` and retrieved from EAV store.
- **Status:** ✅ PASS

## 3. Findings
- **Robust Taxonomy Handling:** Fixed a slug collision bug in `ensure_taxonomy` that was discovered during CLI testing.
- **Managed Area Integrity:** Verified that managed files are correctly streamed to the storage provider and visual proxies are stored in the dedicated `derivatives/` directory.

## 4. Conclusion
The backend is stable and verified. The CLI is a reliable tool for ongoing DX and QA.
