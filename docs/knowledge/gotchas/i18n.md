# i18n (frontend)
Icon: tabler:language
Tags: i18n, localization, svelte
Date: 2026-01-07
Summary: Lightweight translation system backed by a JSON message catalog and a locale-aware helper.

- Source of truth: `frontend/src/core/i18n/messages.json` (all locales + keys).
- Runtime helper: `frontend/src/core/i18n.svelte.ts` (locale resolution + `t()` interpolation).
- Types + locale list: `frontend/src/core/i18n/messages.ts` (`SUPPORTED_LOCALES`, `LANGUAGE_OPTIONS`).
- Settings key: `settings.language` (defaults to `system`); persisted via `frontend/src/core/settings.svelte.ts`.
- System locale is used when available; otherwise the app falls back to English.

## Adding a new translation key
1. Add the key to the `en` block in `frontend/src/core/i18n/messages.json`.
2. Add the same key for every locale block.
3. Run `pnpm -C frontend i18n:check` to verify parity.

## Adding a new language
1. Add a new locale block to `messages.json` (BCP-47 base tag like `it`).
2. Update `LANGUAGE_OPTIONS` in `frontend/src/core/i18n/messages.ts` to expose it in Settings.
3. Optional: update the Settings UI ordering (same file).
4. Run `pnpm -C frontend i18n:check` and `pnpm -C frontend test:e2e` (language tests auto-iterate supported locales).

## Notes
- `LANGUAGE_OPTIONS` drives the Settings > Preferences language buttons.
- The i18n check runs automatically before `pnpm -C frontend build`.
