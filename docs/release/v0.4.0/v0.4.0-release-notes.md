# Release Notes: v0.4.0 (Roadmap)

## Summary
The v0.4.0 cycle is dedicated to cross-platform portability, specifically the delivery of the Mirror Web View thin client. It also serves as the execution phase for the backlog of enhancements identified during the rapid iteration of v0.3.0.

## Key Milestones (Roadmap)
- **Mirror Web View (DEV-5439):** Port the Media Library UI to a browser-ready thin client, verifying VFS portability and platform-agnostic shims.
- **Portability Audit:** Ensure all core services (Settings, Auth, VFS) function correctly outside the Tauri runtime.

## Audit Trail (Active Tasks)
Refer to the categorized folders in this directory (`add/`) for the granular process documentation:
- **DEV-5439:** Mirror Web View (Discovery)

## Backlog & Rollover Tasks (From v0.3.0)
The following items were identified during the v0.3.0 cycle and are queued for implementation in this or future releases:

### Frontend & UX
- [ ] **Sidebar Shortcuts:** Add keyboard shortcuts (e.g., `[` and `]`) to toggle Filter/Tree and Inspector rails.
- [ ] **Capabilities Badge:** Show supported file types and limits in the Prompt Editor UI based on the selected model.
- **Ingestion UX:** Add a "Cancel Batch" button to the progress notification.

### Performance & Durability
- [ ] **ApiExplorer Caching:** Cache model capability results in SQLite to prevent API rate limits.
- [ ] **Ingestion Persistence:** Persist the client-side ingestion queue to SQLite to survive app reloads.
- [ ] **CLI Pagination:** Add pagination flags to `mm-cli vfs list` for large libraries.

### Media Intelligence
- [ ] **Smart Collections:** Dynamic collections based on metadata queries (e.g., "Rating > 4").
- [ ] **Vision Analysis:** Implement `system:color` extraction and OCR text indexing.
- [ ] **Auto-Resize:** Automatically resize images to meet model constraints during prompt execution.

### Architecture
- [ ] **Provider Extraction:** Move OpenAI/R2 specific logic into dedicated `providers/` sub-modules.
- [ ] **GraphQL Standards:** Formalize the "Vertical Slice" directory pattern for all new domains.
